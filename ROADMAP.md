# RAIV for mac Roadmap

この文書は、公開後の機能追加と修正をユーザー価値単位で管理します。実装中の詳細はGitHub IssueとDraft Pull Requestで追跡します。

## P0: 画質設定を分かりやすくする

追跡: [Issue #2 原画＋3種類の簡単画質モード](https://github.com/jydie5/RAIVformac/issues/2)

状態: 実装済み（PR #5でmainへ反映）

通常の読書では「原画」と3種類の補正モードだけを表示します。名称は目視評価後に確定します。

| モード案 | 初期設定 | 目的 |
|---|---|---|
| 原画 | 補正なし | 元画像をそのまま表示・比較 |
| 自然 | Real-CUGAN se / noise 0 | 線とトーンを自然に整える標準設定 |
| クリーニング | Real-CUGAN se / noise 3 | 古いスキャンや圧縮荒れを強めに抑える |
| 高画質 | Real-CUGAN pro / noise 3 | 処理時間より仕上がりを優先 |

旧「軽量」は「標準補正」と同じ処理設定だったため廃止しました。

### 完了条件

- 初めての利用者がモデル名、noise、tileを理解しなくても画質を選べる
- 選択中のモードと原画比較状態が読書画面から判別できる
- 既存の補正結果と見開き単位の同時切り替えを維持する

## P1: マニュアルモードと設定保存

追跡: [Issue #3 マニュアル画質モードとカスタム設定保存](https://github.com/jydie5/RAIVformac/issues/3)

状態: 実装済み（PR #5でmainへ反映）

詳細設定を必要とする利用者だけがマニュアルモードへ切り替えられるようにします。

- モデル、倍率、noise、tile、TTA、補正スキップ解像度を編集
- 名前を付けてカスタム設定を保存、呼び出し、上書き、削除
- 利用できないモデルとパラメータの組み合わせを保存前に検証
- 設定はユーザー領域へ保存し、アプリ更新後も維持
- 簡単モードへ戻してもカスタム設定を消さない

## P1: 気づかせない自動先読み

追跡: [Issue #4 読書速度に追従するバックグラウンド先読み](https://github.com/jydie5/RAIVformac/issues/4)

状態: 実装済み（PR #5でmainへ反映）

読書操作を優先し、通常は先読み処理の存在を意識させない設計にします。

- 表示中の見開き、直後の見開き、遠い前方、後方の順に処理
- 基本値は前方12ページ、後方4ページ
- ページ送り間隔と実測補正時間から前方12〜24ページを自動調整
- 方向転換時は実行中の処理を破棄せず、未着手の対象外ページだけを飛ばして現在位置へ追従
- 原画は即時表示し、左右の補正がそろった時点で見開き単位に切り替え
- PNG読込、グレースケール変換、画面サイズへの縮小は表示スレッド外で実行
- 読書位置は連続操作をまとめ、操作停止後または終了時に保存
- CPU/GPU負荷、メモリ、ディスク使用量に上限を設ける
- 通常時は詳細な処理メッセージを隠し、エラー時だけ簡潔に通知

## P2: 性能評価

- M1、M4 Proで同じ入力を使い、1ページ処理時間と連続ページ送りを測定
- 代表解像度ごとに原画、se、pro、TTAを比較
- キャッシュに追いついた場合の待ち時間と原画表示へのフォールバックを測定
- 10分以上の連続読書でメモリとキャッシュ容量が上限内に収まることを確認

M4 Pro実測（15〜19MBの補正版PNG、1600×1000ウィンドウ）:

- キー入力処理: 0.68ms
- キャッシュ未命中時の原画見開き表示: 15.57ms
- バックグラウンドでの補正版見開き完成: 181.64ms
- 補正版予熱済みのページ送り: 0.57ms

## 完了済み

- 本棚への複数アーカイブ登録、表紙表示、削除
- 右綴じ見開き、1ページずらし、次巻移動、全画面表示
- Real-CUGANの自動補正と原画比較
- 見開き中央寄せと全画面時のステータス行非表示
- 見開き片側だけ補正される問題の防止
- 現在位置周辺だけを保持する補正・表示キャッシュ
- 原画＋3補正のかんたんモード
- マニュアルモードと名前付きカスタム設定
- 読書速度に応じた前方12〜24ページの適応先読み
- 表示デコードの非同期化と100ms以内の原画フォールバック
